# v0.3.0 · Chrome extension + theme shop — EXPLORE brief

> *Stit'Claude · Phase 1. Written before a line of code.*

## Architecture pivot — 2026-04-21

> **Model C adopted. DRY at the highest level.**
>
> Earlier drafts of this brief assumed a *"vanilla JS content script tracker + custom bridges for annotations in v0.4"* split. Wrong move. The right Stit'Claude answer is **dual-instance** : run the lib's React primitives in **two contexts** — the sidePanel AND the host page (via Shadow DOM injection from the content script) — and sync state through a swappable `PersistenceAdapter`.
>
> Zero primitive recoded. `<SelectionToolbar>`, `<NoteEditor>`, `<LinkReveal>`, `<Tooltip>`, `<PinToast>`, `<KeyboardShortcuts>`, `useAttentionTracker` — **tels quels, dans les deux contexts**. Bugs fixés dans la lib profitent aux deux instances automatiquement.
>
> **Le vanilla tracker content.ts est déprécié et sera supprimé** — replacé par un bundle React + lib mounted in Shadow DOM. Ce changement matérialise l'intuition "il suffit de wrapper" que l'utilisateur a portée depuis le début, et qui était juste.

## Architecture — Chrome sidePanel (Chrome 114+)

Le **Chrome sidePanel API** est exactement ce qu'il nous faut. L'UI complète (Typewriter tabs + graph + shop) vit dans le side panel **à côté** de la page — pas injectée dedans. Le content script ne fait QUE le tracking. Séparation propre, zéro styling conflict, UI persistante entre tabs.

```
┌─────────────────────────────────────────────────────────────────┐
│  Browser window                                                 │
│                                                                 │
│   ┌────────────────────────────┐  ┌──────────────────────────┐ │
│   │                            │  │  chrome.sidePanel        │ │
│   │  Current tab content       │  │                          │ │
│   │  (any website)             │  │  ┌──────────────────┐    │ │
│   │                            │  │  │ Typewriter tabs  │    │ │
│   │  ← content-script          │  │  └──────────────────┘    │ │
│   │    tracks attention,       │  │  ┌──────────────────┐    │ │
│   │    stamps [data-mg-id]     │  │  │                  │    │ │
│   │    on observable           │  │  │  Graph 2D OR     │    │ │
│   │    elements, sends         │  │  │  Constellation   │    │ │
│   │    commits to worker       │  │  │                  │    │ │
│   │                            │  │  └──────────────────┘    │ │
│   │                            │  │  ┌──────────────────┐    │ │
│   │                            │  │  │ Shop trigger     │    │ │
│   │                            │  │  │ (top-satellite)  │    │ │
│   │                            │  │  └──────────────────┘    │ │
│   └────────────────────────────┘  └──────────────────────────┘ │
│                ↕ chrome.runtime.sendMessage                     │
│   ┌─────────────────────────────────────────────────────────┐  │
│   │  Background service worker (non-persistent)             │  │
│   │                                                         │  │
│   │  · chrome.storage.local = unified super-state           │  │
│   │  · route = pathname, site = origin                      │  │
│   │  · broadcasts updates to sidePanel + content scripts    │  │
│   │  · owns theme license gate (offline-first)              │  │
│   └─────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

**Pourquoi sidePanel > content-script inject UI :**
- **Zero styling conflict** avec les CSS du site visité (Shadow DOM inutile)
- **Persistance cross-tab** : le panel reste ouvert quand l'user switch d'onglet, idéal pour notre ambient philosophy
- **Full Chrome API access** (même contexte qu'une page d'extension)
- **Standard MV3** (Chrome 114+)

**Storage key hierarchy** : `mg:super` = root. Nodes tagged `{ route, site }`. Le lib data model existe déjà — besoin juste d'un `chrome.storage` adapter qui mappe vers `SerializedGraph`.

## Manifest MV3 minimum

```json
{
  "manifest_version": 3,
  "name": "memory-graph",
  "version": "0.3.0",
  "description": "Your reading, visualized. Across every site.",
  "icons": { "48": "icon-48.png", "128": "icon-128.png" },
  "permissions": ["sidePanel", "storage", "tabs"],
  "host_permissions": ["<all_urls>"],
  "background": { "service_worker": "service-worker.js" },
  "content_scripts": [{
    "matches": ["<all_urls>"],
    "js": ["content.js"],
    "run_at": "document_idle"
  }],
  "side_panel": { "default_path": "sidepanel.html" },
  "action": { "default_title": "Open memory-graph" }
}
```

`action.default_title` + `chrome.sidePanel.setPanelBehavior({openPanelOnActionClick: true})` → click de l'icône toolbar ouvre le sidePanel. Pas de popup intermédiaire.

## Scope v0.3.0 (in)

1. **Extension skeleton** — manifest MV3, service worker (non-persistent), content script (tracking-only), sidepanel.html + React entry, icons 48/128
2. **sidePanel setup** — `setPanelBehavior({openPanelOnActionClick: true})`, `setOptions({path, enabled})` par tab si besoin
3. **Multi-origin bridge** — content scripts envoient commits via `chrome.runtime.sendMessage` → service worker écrit dans `chrome.storage.local` → broadcast `chrome.runtime.sendMessage` vers le sidePanel (qui est un page, écoute `chrome.runtime.onMessage`)
4. **Typewriter tabs** dans le sidePanel — site switcher aux 4 états (rest/switch/pulse/hover)
5. **Constellation view** — vue alternative `∑ all`, super-nodes par site + dashed arcs inter-sites
6. **Theme shop modal** — déclenchée par le top-satellite, galerie de thèmes, preview live sur le graph courant, bouton "acheter/activer"
7. **2–3 thèmes de lancement** — `stit-claude` (default free) + 2 premium (suggestions : **Cyberpunk** — grille + mono distopique + accent néon vert, **Japanese-Ma** — minimal + beige papier + accent rouge vermillon)
8. **License gate** — token stocké dans `chrome.storage.local` par thème acheté ; offline-friendly (pas de re-check serveur à chaque load)

## Scope hors v0.3.0 (deferred)

- **Cross-route click** — cliquer un node qui appartient à une autre route/site → navigate + scroll + highlight (v0.3.1 ou fin v0.3.0 si temps)
- **Cross-route hover** — hover un link dont la cible est ailleurs → arc visuel vers la colonne/tab cible (v0.3.1)
- **Bidirectional hover** paragraphe ↔ node graph cross-context (v0.4.1)
- **Click node → scroll page** cross-context message bridge (v0.4.1)
- **Content-density auto-tag** — tagger les div wrappers de contenu lisible (Notion, Twitter, etc) via heuristique density + exclude list (nav/header/footer/aside/role="navigation"). User-initiated mark "Track this block" via right-click (v0.4.1)
- **5 autres thèmes** (Editorial, Swiss-Quiet, Apple-Craft, Industrial-Pro, Playful-Warm) → v0.3.x drip release
- **Firefox port** → v0.3.x une fois Chrome stable
- **Privacy controls** (blocklist sites, incognito mode, clear-all) → v0.3.1

**Specs complètes** (à implémenter en Phase 3) :
- Tile = 32×32 squircle, rayon 6px (concentrique avec les autres squircles du composant)
- Favicon 16×16 natif centré, ou **lettre-initiale en Fraunces italic** comme fallback
- Border : hairline 0.5px, opacity muted
- Coral bar baseline : 2px × 20px, centré sous la tile active, `transform: translateX` spring-snappy 280ms expo-out (**même token que les zoom icons**)
- Baseline des tiles : dashed 0.5px opacity 0.3 pour inactives, solide coral pour active
- Activity pulse : dashed coral halo border-style sur la tile, 500ms ease-out, déclenchée par un message `{ type: 'activity', siteId }` du service worker
- `∑ all` tab séparée par gap 12px + vertical hairline 1px, sigma glyph en mono font (pas de favicon)
- Order dynamics : sites récents glissent à gauche avec un reflow spring (FLIP technique)
- Règle absolue : **ne JAMAIS coloriser les tiles avec les couleurs natives du site** (favicon = contenu, pas chrome — même règle que `<img>` dans un article editorial)

## Token system — DISTILL (preview)

Aucun nouveau token pour le chrome de l'extension. **Chaque thème payant** apporte son propre set override :

```css
[data-mg-theme="cyberpunk"] {
  --mg-accent: oklch(0.82 0.21 145);   /* néon vert */
  --mg-bg:     oklch(0.12 0.02 260);   /* nuit distopique */
  --mg-fg:     oklch(0.92 0.02 145);
  --mg-font-display: "JetBrains Mono", monospace; /* mono everywhere */
  /* ... etc */
}
```

Le base `_defaults.css` reste la source de vérité ; les thèmes ne font qu'**étendre** via sélecteur attribut. Pas de breaking.
